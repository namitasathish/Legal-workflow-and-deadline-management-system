Design Improvements
1
Gradient hero header
Indigo-to-dark-slate gradient replaces flat white. Adds brand personality and depth immediately on open.
2
Branded logo mark
Frosted glass app icon with a document grid glyph. Consistent brand identity users associate with trust.
3
Role removed from login

4
"Forgot Password?" link
Non-functional placeholder that signals app maturity. Users expect it — absence feels unfinished.
5
Refined input fields
Subtle fill, rounded corners, uppercase spaced labels — professional legal aesthetic, not generic.

Design Improvements
1
Dark gradient case header
Priority badge, court, filing date, and urgent hearing pill all live in a compact dark hero — dramatic and scannable.
2
Horizontal tab bar
Overview | Tasks | Documents | Payments | Chat replaces 615 lines of scroll. Each tab owns its content — zero cognitive load.
3
Interactive timeline
Colour-coded dots: purple = done, amber = current, gray = future. Lawyer and client both read progress immediately.
4
Collapsible sections
Each card has a ▾ toggle so sections can be collapsed. Reduces visual clutter for power users who know their case.
5
Chat preview in Overview
Last two messages visible on Overview tab with an "Open →" affordance. No need to switch tab just to check latest message.

Design Improvements
1
Teal/green accent system
Client portal is immediately distinct from the lawyer's indigo. Zero ambiguity which portal you're in — critical for demos.
2
Active case hero card
Glassmorphism card inside the header shows court, case name, hearing date and priority — clients see the most important info first.
3
Document upload center
Pending docs shown as action items with teal "Upload" CTAs. Count badge on the section header drives urgency without panic.
4
Simplified bottom nav
Clients get 4 focused tabs (Home, My Case, Chat, Profile) instead of the full lawyer nav — right-sized for their use case.
5
Teal timeline accent
Same timeline component, teal instead of indigo for the current milestone — visually consistent with the portal identity.

Empty State Philosophy
1
Context-specific icons
Scales-of-justice for cases, speech bubble for chat, folder for docs. Each empty state feels deliberate, not like a generic stub.
2
CTA inside empty state
Every empty state has a colored action button. Users are never stranded — they know exactly what to do next.
3
Color-matched CTAs
Each section's CTA uses the section's own accent color — Cases = indigo, Docs = amber, Billing = blue.
